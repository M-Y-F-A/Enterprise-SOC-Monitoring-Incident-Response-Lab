# Detection & Alerts Dashboard

## Overview

The Detection & Alerts Dashboard provides SOC analysts with a centralized view of security alerts generated by Splunk detection rules. It enables rapid identification, prioritization, and investigation of suspicious activities across the enterprise environment.

This dashboard supports security monitoring, incident triage, and validation of implemented detection rules.

---

# Purpose

- Monitor active security alerts.
- Prioritize incidents based on severity.
- Track detection rule activity.
- Identify affected hosts and user accounts.
- Support rapid incident triage and response.

---

# Data Sources

- Splunk Alerts
- Windows Security Log
- Sysmon
- Detection Rules

---

# Dashboard Panels

## Active Alerts

**Purpose**

Display all active security alerts generated by Splunk detection rules.

---

## Alerts by Severity

**Purpose**

Visualize the distribution of alerts by severity level.

---

## Alerts by Detection Rule

**Purpose**

Display the number of alerts generated by each detection rule.

---

## Alert Timeline

**Purpose**

Visualize alert activity over time to identify spikes and ongoing attack campaigns.

---

## Top Alerted Hosts

**Purpose**

Identify endpoints generating the highest number of security alerts.

---

## Top Alerted Users

**Purpose**

Display user accounts most frequently associated with security alerts.

---

## Recent Critical Alerts

**Purpose**

Highlight recently generated Critical and High severity alerts requiring immediate investigation.

---

## Alert Status Summary

**Purpose**

Provide an overview of alert status (New, In Progress, Closed) to support SOC operations.

# Dashboard Validation

The dashboard is considered successfully implemented when:

- Detection alerts are generated correctly.
- Alert severity is displayed accurately.
- All dashboard panels populate with real-time data.
- Alerts can be correlated with their corresponding detection rules.
- Dashboard supports efficient incident triage and investigation.
